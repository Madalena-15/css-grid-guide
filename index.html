Spanning the Items on the Grids

To make a grid item take up more than one column size is to span the grid line number in the grid-column property. For example, to make the first grid item (in red) take up two columns shown below, the grid column line starts at 2, and it ends at 4. The fourth grid item (in pink) will be moved below to the next row.

Use this property - grid-area to either position and span the grid item in both column and row. The order of value for this property goes like this:

grid-area: [row-start] / [column-start] / [row-end] / [column-end];

In the example below, the value of the fifth grid item (in blue) goes like this:
grid-area: 1 /3 / 2 / 4;

There are different ways to set the value to span grid items across the columns and row. For example, the second item (in green) is positioned in the second column and span to the third. It could be written as grid-column: 1/3;

It also could be written as: grid-column: 1/span 2; This means it will span over two columns. It is useful when unsure what grid line it is currently on, but knows how many columns or row to span over.

Another useful way is to write it is grid-column: 1/-1; This will span all the way to the end of the column (or row in other cases). It is good for when unsure how many rows or columns there are.

Positioning the Items by Naming the Areas

Give grid-cells name. It is done in the grid-template-row and grid-template-column properties with the names in the double-quotes (see code for better understanding). Within the double quotes represents either one row or column and the number of values enters in are the number of columns or row.

For example, to make the header span across four columns, in the grid-template-column property and within the double quote, place 'head' four times. Then, add the grid-area property in the .header div with value 'head'. It will then span across four columns.

It goes the same in the second row where there is the .main div and .side div. In the grid-template-column property, there are three 'main' and one 'side', making a total of four columns. When the value 'main' is added in the grid-area property for .main div, it will span across three columns.

Explicit Grids Vs. Implicit Grids

When you generate a set amount of rows or columns and the grid items are set and fit within them, this is explicit grid. But, for example as shown below, if you set two sets of rows and columns, which means it can take the total of four grid items, and you add more than four items. What will happen is the grid container will create grid tracks to add these extra grid items within them. It is known as implicit grids. Using auto-fit & auto-fill for responsive layouts

It is useful for creating responsive layout without using media queries. auto-fill and auto-fit are values used in the grid-template-rows or grid-template-columns properties. The difference bewtween the two values are to do with the column's sizing.

auto-fill is to enable space in the empty column within a row. For example below, there are eight grid items within the 1000px wide grid container. All eight grid items are placed in one row as altogether it is 800px and within the remaining 200px space, it is empty, although technically it does has two columns.
